# Contributing

Thanks for helping.

## How to contribute
- **Bugs**: open an issue with reproduction steps + expected/actual behavior.
- **Features**: open an issue first so we can align on scope.
- **PRs**: keep changes small and focused.

## Local development
```bash
npm ci
npm test
```

## What we care about
- Deterministic output (stable PR comments)
- Low flake rate in Playwright (timeouts, waits, retries)
- Clear docs for pilots

## Code of Conduct
By participating, you agree to follow `CODE_OF_CONDUCT.md`.
